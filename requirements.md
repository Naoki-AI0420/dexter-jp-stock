# 日本株版 Dexter — 要件定義書
> 最終更新: 2026-04-01

## 1. プロダクト概要

### コンセプト
**「聞くだけで、プロの分析が返ってくる」** — 機関投資家がアナリストチームにやらせていることを、AIエージェントが1人でやる。日本の個人投資家に開放する。

### ターゲットユーザー
- 新NISAで株を始めた個人投資家（決算の読み方がわからない）
- 兼業投資家（決算シーズンに全部読む時間がない）
- 中級者（スクリーニングの先の「判断」が欲しい）

### 解決する課題
1. **決算が読めない** — 数字の羅列。何を見ればいいかわからない
2. **銘柄が多すぎて選べない** — 東証3,900社+。スクリーナーの先で止まる
3. **持ち株の「今」がわからない** — 決算出たけど良い？悪い？ホールド？売り？
4. **情報が「点」で「判断」に変わらない** — ニュース・四季報の先がない
5. **機関投資家との武器格差** — Bloomberg年200万 vs Yahoo!ファイナンス

### 提供価値
- AIエージェントが自律的に分析（質問分解→計画→データ収集→分析→検証→回答）
- リアルタイム公式データ（J-Quants API）に基づく根拠ある分析
- 有価証券報告書（EDINET）の全文読解
- 日本語ネイティブ対応

### 重要な制約
- **SaaSとしては分析・提案まで。投資助言・自動売買機能は含めない**
- **料金プランは未確定**（後日AYさんと確定）
- **「投資は自己責任」の免責事項を必ず表示**

## 2. 機能要件

### 2.1 コア機能（MVP — Phase 1）

#### 対話型AI分析エンジン
- ユーザーが日本語で質問 → AIエージェントが分析結果を返す
- Dexterのマルチエージェント構造を踏襲:
  - Planning Agent: 質問を分解し、リサーチステップを計画
  - Execution Agent: 適切なツールを選択・実行してデータ収集
  - Validation Agent: 分析結果をセルフチェック
  - Response Agent: 根拠つきで分かりやすく回答
- 対話履歴の保持（セッション内）

#### データ取得ツール群
- **株価データ**: J-Quants API v2（日足・週足・月足）
- **財務データ**: J-Quants API v2（決算短信、財務諸表）
- **企業情報**: J-Quants API v2（銘柄一覧、セクター、時価総額）
- **有価証券報告書**: EDINET API v2（全文XBRL → テキスト変換）
- **ニュース検索**: Web検索（Exa or Tavily）

#### 使用シーン（対応すべきクエリ）
- 「トヨタの今期決算どうだった？」→ 売上・利益・前期比・ガイダンス分析
- 「この銘柄、買い？」→ 財務分析 + バリュエーション + リスク要因の総合判定
- 「PER15倍以下で増収増益の銘柄」→ スクリーニング + 各銘柄の分析
- 「トヨタとホンダ、比較して」→ 財務・成長性・バリュエーション全比較
- 「100万円でポートフォリオ組んで」→ セクター分散 + リスク調整 + 理由つき
- 「決算出たけど良いの？悪いの？」→ 前期比較 + ガイダンス分析 → 即回答

### 2.2 Web UI（Phase 1）
- チャットインターフェース（1画面完結）
- 銘柄検索（ティッカー or 企業名）
- 分析結果のリッチ表示（テーブル、チャート、ハイライト）
- レスポンシブ対応（スマホ重要 — 通勤中に使う）
- ダークモード

### 2.3 認証・課金（Phase 2）
- メール+パスワード認証（NextAuth.js）
- Google OAuth
- Stripe決済連携
- プラン管理（Free / Pro / Premium — 料金未確定）
- 利用回数制限（プランごと）

### 2.4 拡張機能（Phase 2-3）
- 分析結果のPDF出力
- ウォッチリスト（銘柄お気に入り）
- 決算カレンダー（持ち株の決算日一覧）
- 分析結果の共有リンク
- LP（ランディングページ）

## 3. 非機能要件

### パフォーマンス
- 分析レスポンス: 30秒以内（複雑なクエリでも）
- UI初期表示: 3秒以内
- API応答: 500ms以内（キャッシュヒット時）

### セキュリティ
- APIキーはサーバーサイドのみ（クライアントに露出しない）
- CORS明示的オリジン指定
- レート制限（DDoS防止）
- ユーザーデータ暗号化

### スケーラビリティ
- 初期: 同時100ユーザー対応
- Docker Compose → 水平スケール可能な設計

### 法的要件
- **免責事項**: 「本サービスは情報提供のみを目的としており、投資助言ではありません」
- **金融商品取引法**: 投資助言業の登録なしで運営可能な範囲に留める
- **J-Quants API利用規約**: データの再配布禁止。分析結果の提供はOK
- **特定商取引法に基づく表記**
- **プライバシーポリシー**

## 5. データソース詳細

### J-Quants API v2（JPX公式）
- **提供元**: JPX Market Innovation & Research
- **エンドポイント**: 株価（日足/分足/ティック）、財務諸表、企業情報、指数、決算カレンダー、信用取引残高
- **認証**: APIキー（ダッシュボードで発行）
- **プラン**: Free(¥0) / Light(¥1,650) / Standard(¥3,300) / Premium(¥16,500)
- **推奨**: Standard(¥3,300/月) — 全銘柄の財務データ+日足が取得可能
- **制約**: データの再配布禁止。分析結果の提供はOK
- **2026年1月更新**: CSV配信追加、分足・ティックデータ追加

### EDINET API v2（金融庁）
- **提供元**: 金融庁
- **エンドポイント**: 提出書類一覧、書類取得（XBRL/PDF）
- **認証**: APIキー（要申請）
- **料金**: 無料
- **取得可能データ**: 有価証券報告書、四半期報告書、決算短信等
- **制約**: XBRL解析が必要（名寄せが複雑 — JP GAAP/IFRS/US GAAPで項目名が異なる）
- **参考**: edinet-tools（Python SDK）、edinetdb（名寄せ済みAPI）

## 6. Dexterからの変更点

| 項目 | 本家Dexter | 日本株版 |
|---|---|---|
| データソース | Financial Datasets API（米国株） | J-Quants API + EDINET（日本株） |
| 対象市場 | 米国株（NYSE/NASDAQ） | 日本株（東証プライム/スタンダード/グロース） |
| 言語 | 英語 | 日本語 |
| Filing | SEC（10-K/10-Q/8-K） | EDINET（有価証券報告書/四半期報告書） |
| 通貨 | USD | JPY |
| 指標 | S&P 500, NASDAQ | 日経225, TOPIX |
| UI | ターミナル（CLI） | Web UI（Next.js） |
| 認証+課金 | なし（OSS） | NextAuth + Stripe |
| デプロイ | ローカル | Docker → VPS |
